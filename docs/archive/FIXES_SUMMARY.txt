MIGRATION WORKFLOW
═════════════════════════════════════════════════════════════════════════

Development (local):
  python manage.py makemigrations     # Create migrations on host
  python manage.py migrate            # Apply locally
  python manage.py test               # Test your changes

Deployment (Docker):
  ./build.sh                          # Builds images and applies migrations

Key Point: Migration files MUST be committed to git. When others run
./build.sh, migrations will be automatically applied from the committed files.
